# Pull Request: End-to-end LLM-сервис с CI, конфигурацией и PR

## 📋 Описание изменений

Создание полноценного end-to-end LLM-сервиса, соответствующего требованиям production-like решений. Сервис обрабатывает пользовательские запросы с использованием GitHub Models API, поддерживает кеширование, обработку ошибок и fallback-ответы.

## 🏗️ Архитектура решения

### Основные компоненты

1. **API Layer (`api/routes.py`)**
   - FastAPI роуты для HTTP API
   - Эндпоинт `POST /api/v1/process` для обработки текста
   - Эндпоинт `GET /health` для проверки работоспособности
   - Валидация входных данных через Pydantic модели

2. **LLM Module (`llm/`)**
   - `client.py` — клиент для GitHub Models API с обработкой ошибок и таймаутов
   - `prompt_builder.py` — формирование системных и пользовательских промптов для 3 сценариев

3. **Business Logic (`services/processor.py`)**
   - Центральный обработчик текста
   - Интеграция кеширования, LLM-клиента и пост-обработки
   - Fallback-ответы при недоступности LLM

4. **Configuration (`config/settings.py`)**
   - Настройки через переменные окружения
   - Разделение dev/prod конфигураций
   - Отсутствие хардкода секретов

5. **Caching (`cache/cache.py`)**
   - InMemoryCache с TTL
   - Хеш-ключи на основе текста + сценария
   - Логирование cache hit/miss

6. **Testing (`tests/`)**
   - 19 тестов покрывающих все компоненты
   - Тесты API, кеша и LLM-модуля
   - CI-пайплайн с flake8 и pytest

## 📦 Сценарии работы

1. **business_letter** — формирование делового письма
2. **social_post** — создание поста для соцсетей
3. **creative** — генерация креативных идей
